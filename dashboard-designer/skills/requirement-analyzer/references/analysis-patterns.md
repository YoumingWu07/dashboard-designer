# 需求分析模式参考

本文档定义了BI驾驶舱需求分析的常用模式和方法。

## 需求提取框架

### 5W1H分析法

在分析需求时，关注以下要素：

| 要素 | 问题 | 示例 |
|------|------|------|
| What | 做什么功能？ | 销售数据展示看板 |
| Who | 谁使用？ | 销售经理、运营人员 |
| When | 什么时候用？ | 每日晨会、月度复盘 |
| Where | 在哪里用？ | 办公室电脑、移动端 |
| Why | 为什么需要？ | 实时掌握销售动态 |
| How | 怎么用？ | 查看KPI、下钻分析 |

## 需求分类

### 功能需求

1. **展示类需求**
   - KPI指标展示
   - 趋势图表展示
   - 数据表格展示
   - 地图展示

2. **交互类需求**
   - 筛选过滤
   - 下钻分析
   - 联动刷新
   - 导出下载

3. **配置类需求**
   - 主题切换
   - 指标配置
   - 告警设置

### 非功能需求

1. **性能需求**
   - 页面加载时间 < 3秒
   - 数据刷新频率
   - 并发用户数

2. **安全需求**
   - 数据权限控制
   - 操作审计日志
   - 敏感数据脱敏

## 需求优先级

| 优先级 | 说明 | 示例 |
|-------|------|------|
| P0 | 核心功能，必须实现 | 主要KPI展示 |
| P1 | 重要功能，应该实现 | 下钻分析 |
| P2 | 增强功能，可以实现 | 数据导出 |
| P3 | 优化功能，暂缓实现 | 主题切换 |

## 需求澄清清单

在分析需求时，确保回答以下问题：

1. [ ] 项目背景是否清晰？
2. [ ] 目标用户是否明确？
3. [ ] 核心功能是否定义？
4. [ ] 数据来源是否确定？
5. [ ] 交互方式是否描述？
6. [ ] 权限要求是否明确？
7. [ ] 性能要求是否量化？
8. [ ] 验收标准是否制定？
