# 图片解析指南

本文档指导如何从图片中提取BI看板的设计信息。

## 图片类型识别

### 1. 手绘草图

**特征**：
- 线条简单、可能有涂改痕迹
- 布局结构清晰
- 文字可能不规整

**提取重点**：
- 页面布局结构
- 组件位置关系
- 功能区域划分

### 2. 系统截图

**特征**：
- 完整的UI界面
- 包含真实数据
- 交互元素清晰

**提取重点**：
- 组件类型识别
- 数据字段推断
- 交互方式识别

### 3. 参考设计图

**特征**：
- 高保真设计稿
- 配色方案完整
- 样式细节丰富

**提取重点**：
- 配色方案
- 组件样式
- 字体规范

### 4. 白板/会议照片

**特征**：
- 可能有多个人的字迹
- 可能有箭头、连线
- 背景可能有干扰

**提取重点**：
- 关键文字内容
- 流程和关系
- 结构框架

## 解析流程

### 步骤1: 整体布局识别

识别图片的整体结构：

```
┌─────────────────────────────────────┐
│ 识别页面区域：                       │
│ - 顶部区域：标题、筛选器、操作按钮   │
│ - 左侧区域：导航、筛选面板           │
│ - 主内容区：图表、表格               │
│ - 右侧区域：辅助信息                 │
│ - 底部区域：分页、备注               │
└─────────────────────────────────────┘
```

### 步骤2: 组件类型识别

识别各区域的组件类型：

| 视觉特征 | 组件类型 |
|---------|---------|
| 大数字 + 小标签 | KPI指标卡 |
| 坐标轴 + 线条/柱状 | 图表(折线/柱状) |
| 圆形/环形切片 | 饼图/环形图 |
| 行列结构 | 数据表格 |
| 地图轮廓 | 区域地图 |
| 下拉箭头 | 筛选器 |
| 可点击按钮 | 操作按钮 |

### 步骤3: 文字内容提取

提取关键文字信息：

- **标题文字**: 页面标题、组件标题
- **标签文字**: 指标名称、维度名称
- **数值文字**: 具体的数据值（推断数据格式）
- **说明文字**: 图例、备注

### 步骤4: 数据推断

基于组件和文字推断数据需求：

```
KPI卡片 "销售额 ¥128.5万"
→ 推断: 需要销售额字段，支持金额格式化

趋势图 "6个月销售趋势"
→ 推断: 需要月度销售数据，时间维度

饼图 "区域销售占比"
→ 推断: 需要按区域汇总的销售数据
```

### 步骤5: 交互识别

识别交互元素：

- **筛选器**: 下拉框、日期选择、单选/多选
- **按钮**: 刷新、导出、设置
- **可点击区域**: 下钻入口、跳转链接
- **联动元素**: 选中状态、高亮显示

## 输出格式

解析完成后，输出结构化信息：

```markdown
## 图片分析结果

### 1. 布局结构
- 整体布局: [3行布局/左右分栏/大屏布局]
- 页面区域: [列出各区域及其内容]

### 2. 组件清单
| 位置 | 组件类型 | 标题/内容 | 推断数据 |
|------|---------|----------|---------|
| 第一行左1 | KPI卡 | 销售额 | sales_amount |
| ... | ... | ... | ... |

### 3. 筛选器
| 筛选器名称 | 类型 | 选项/默认值 |
|-----------|------|------------|

### 4. 配色方案 (如果是参考设计图)
- 主色: #XXXXXX
- 辅色: #XXXXXX
- 背景: #XXXXXX

### 5. 交互说明
- [描述识别到的交互行为]
```

## 注意事项

1. **模糊处理**: 图片不清晰时，标注为"待确认"
2. **文字识别**: 识别困难的文字，结合上下文推断
3. **数据推断**: 基于组件类型合理推断数据需求
4. **存疑标注**: 不确定的信息标注出来，与用户确认
